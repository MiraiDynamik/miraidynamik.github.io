How to Use Quartus Prime Lite 19.1 and ModelSim Altera to Create and Simulate a Circuit

Introduction

These instructions teach complete beginners how to use Quartus Prime Lite 19.1 and ModelSim Altera by guiding them through creating and simulating a full adder circuit. In the next section, you will implement this full adder on a Intel DE-10 Lite FPGA Board, later in this series, you will learn about incorporating this full adder in an arithmetic logic unit, which is a part of a computer. Following these instructions, you will first create a project. Then you will capture a schematic, which means drawing a circuit with abstract symbols that each represents a circuit component; this involves adding inputs, outputs, logic gates, and connecting these components together with wires. Finally, you will simulate this circuit design, which means giving the circuit virtual inputs and observe the outputs.

Figure 1 shows the graphic user interface of Quartus Prime Lite, with a schematic of a full adder in it. Following this tutorial, you will create a schematic file in 'File' under A, the main menu bar, and then save it with B, the save button. After saving the file, the canvas, C will appear in the center of the screen. Double click canvas on any space to create components, then connect components together with D, the Orthogonal Node Tool, which is basically a wire tool, into E, the schematic of a full adder. After editing the schematic, you are going to save it with B, then click F to compile it, wait until successful message appear in G, the message panel. Finally, you will configure for simulation and open ModelSim Altera in 'Tools' under A.

Figure 2 shows the simulation window of ModelSim Altera. Following this tutorial, you will first start the simulation with 'Simulate' under A, then B, the objects panel will appear. You are going to find input and output of the circuit in B, and add them to C, the waves window, so that these objects will display in D. You will then run the simulation and the results will show up in E, if the results align with expected values, you have successfully created a working full adder.

Steps 1-10

Step 1 -- Open Quartus Prime and Create a Project

Start Quartus Prime, you will see figure 4, click 'New Project Wizard'

You will see figure 5a, where A, the new project wizard window appears. Click next.

Now you can see figure 5b, set the project directory as a path without a space, name the project as FullAdder. Click 'Next'.

Click 'Next' until you can see figure 5c: Family, Device & Board Settings Select the Family of device as DE-10 at B, scroll down through C, the list of Avaliable devices, until you find 10M50DAF484C7G. Select 10M50DAF484C7G.

Click next, you will see figure 5d, select ModelSim Altera as the gate level simulation tool at D, select the format as Verilog HDL. Click 'Next' and then click 'Finish', and you have successfully created a project.

Caution: Do not select ModelSim as the EDA tool for simulation, since it is different with ModelSim Altera that we are going to use.

Step 2 -- Create a Schematic File

Now you can see this interface as figure 6a, click 'File' and 'New' as 6b shows, then you will see the 'New' window like figure 6c, select 'schematic/ block diagram' and click 'OK'. Save the file.

Step 3 -- Place Elements and Connect Wires

You have created a empty schematic like figure 7, double click anywhere empty on the canvas.

Then you can see a pop up like figure 8, where you can search for elements. Type 'input' in A, select 'Repeat-insert mode', click 'OK', and click anywhere empty on the canvas for 3 times to create 3 input elements.

Now repeat to add new elements, search for 'output', and create 2 output elements, search for 'or3' and create a three-input or gate, search for 'and2' and create 3 two-input and gates, search for 'xor' and create 2 two-input xor gates.

Double click the name of any symbol to rename it like figure 9 shows. Rename the three inputs exactly as 'SW[0]', 'SW[1]', and 'SW[2]'; rename the two outputs exactly as 'LEDR[0]' and 'LEDR[1]'

Drag symbols around to arrange them roughly like what figure 10 shows.

Finally, select the Orthogonal Node Tool and draw wires like figure 11, and you have completed building the schematic of a full adder.

Caution: Drag symbols around before (but not after) wires are attached to them, since different wires often stick with each other or with symbol if they meet each other when you are moving them.

Step 4 -- Compile the Circuit

Save your changes and click compile like figure 12, the compliation report view A will appear, wait for the compilation to complete. When the compilation is complete, the message window B will show a message like "Info (293000): Quartus Prime Full Compilation was successful. 0 errors,14 warnings". You don't have to do anything if there are warnings.

Step 5 -- Configure for Simulation

Like in figure 13, click 'tools' and click 'options' in the dropdown, then you will see the 'Options' window, click 'EDA Tool Options', and you will see figure 15. Insert the path to modelsim.exe in A, the entry 'ModelSim Altera', click 'OK'

Step 6 -- Open ModelSim Altera (Through Quartus Prime)

Like figure 14, click 'tools', hover on 'run simulation tool', and click 'gate level simulation', so that ModelSim Altera will start.

Like figure 15, you will see the graphic user interface of ModelSim Altera, with A. Menu bar, B. Objects panel, and C: Wave window

Step 7 -- Configure for and Start the Simulation

Click 'Simulate', then click 'start simulation' in the dropdown, you will see a window like figure 16

Like figure 16a, click the '+' beside 'work' to expand, and click 'FullAdder' under 'work'. Then click 'Libraries' in the same Start Simulation window, and you will see the page in figure 16b.

Like figure 16b, click 'add'. Window A will appear, click the triangle on the left of 'Browse...' to show all libraries, select 'altera_ver', and click 'OK'. Repeat the process above to add 'fiftyfivenm_ver', you will see figure 16c

Click 'OK', A: 'Objects' window will fill up by the objects from your schematic like figure 17

Step 8 -- Forcing Inputs and What to Expect

Like figure 18, right click 'LEDR' under Objects, click 'Add Wave', and repeat for 'SW'.

What you will see is like figure 19, objects LEDR and SW appears in the wave window, now right click 'SW', click 'force' in the dropdown.

'Force Selected Signal' window will appear, figure 21 shows the 'Force Selected Signal' window. Insert '000' into A: 'Value', as the input. Press key f9, so the simulation will progress for 100 ps, which should be the default run length, and the expected output is '00', like figure 22.

Try forcing all the inputs one by one on the left side of the truth table as figure 23, for example, the input of second row is '001', and the expected output is '01' Remember to press f9 key every time you modify the input.

After simulating all eight inputs, you can maximize the wave window and adjust the view to see the full waveform. Now you are likely to see what figure 24 shows. Compare the A: outputs and B: inputs with the truth table in figure 23. If all the outputs corresponds to the right side of the truth table, your full adder is working as intended!

Conclusion

Now you have learnt how to create a logic circuit with Quartus Prime and simulate its behavior under inputs using ModelSim Altera. In the next section, you will learn about implementing this full adder on a Intel DE-10 Lite FPGA Board.
